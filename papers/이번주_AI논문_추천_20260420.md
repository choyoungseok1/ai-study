## 1. Understanding Transformers and Attention Mechanisms: An Introduction for Applied Mathematicians

**arxiv**: [https://arxiv.org/abs/2604.00965](https://arxiv.org/abs/2604.00965)
**발표일**: 2026년 4월 1일
**코드**: 없음 (개념 설명 중심)

### 초록 요약
Transformer 아키텍처의 핵심인 Attention 메커니즘을 수학적 관점에서 소개하는 논문. 텍스트가 벡터로 인코딩되는 과정, Multi-Headed Attention의 작동 원리, Transformer 아키텍처의 변형, 그리고 KV caching, Grouped Query Attention, Latent Attention 등 최신 효율화 기법까지 다룬다.

### 추천 이유
1. 지금 진행 중인 Transformer vs Seq2Seq 비교 프로젝트에 직접적으로 도움이 되는 논문으로, Attention 메커니즘의 수학적 직관을 쌓을 수 있다.
2. 2026년 4월 최신 논문이라 KV caching, GQA 등 현재 LLM에서 실제로 쓰이는 최적화 기법까지 커버한다.
3. "입문자를 위한 소개"를 표방하므로, Python 기초를 막 마친 단계에서도 충분히 소화 가능한 난이도다.

## 2. Paper2Code: Automating Code Generation from Scientific Papers in Machine Learning

**arxiv**: [https://arxiv.org/abs/2504.17192](https://arxiv.org/abs/2504.17192)
**발표일**: 2025년 4월 (2026년 2월 업데이트)
**코드**: ✅ [https://github.com/going-doer/Paper2Code](https://github.com/going-doer/Paper2Code)

### 초록 요약
ML 논문을 읽고 자동으로 실행 가능한 코드 리포지토리를 생성하는 멀티 에이전트 LLM 프레임워크 "PaperCoder"를 제안. Planning(설계도 작성) → Analysis(구현 세부사항 해석) → Generation(모듈별 코드 생성)의 3단계 파이프라인으로, 생성된 코드의 77%가 인간 평가에서 최고 점수를 받았다.

### 추천 이유
1. LLM 응용 엔지니어가 되려는 목표에 딱 맞는 논문으로, "LLM 에이전트가 실제 문제를 어떻게 해결하는지" 구체적 사례를 보여준다.
2. 멀티 에이전트 설계 패턴(Planning → Analysis → Generation)은 현업에서 가장 많이 쓰이는 LLM 앱 아키텍처 중 하나다.
3. GitHub 코드가 공개되어 있어 직접 돌려보며 에이전트 파이프라인 구조를 실습할 수 있다.

## 3. A-RAG: Scaling Agentic Retrieval-Augmented Generation via Hierarchical Retrieval Interfaces

**arxiv**: [https://arxiv.org/abs/2602.03442](https://arxiv.org/abs/2602.03442)
**발표일**: 2026년 2월 3일
**코드**: ✅ [https://github.com/Ayanami0730/arag](https://github.com/Ayanami0730/arag)

### 초록 요약
LLM에 계층적 검색 인터페이스(keyword_search, semantic_search, chunk_read)를 직접 노출하여, 에이전트가 다양한 검색 전략을 자율적으로 선택하도록 하는 Agentic RAG 프레임워크. GPT-4o-mini/GPT-5-mini를 백본으로 사용하여 기존 Graph-RAG, Workflow RAG 대비 일관된 성능 향상을 달성했다.

### 추천 이유
1. RAG는 LLM 응용의 가장 기본이자 핵심 기술로, ML 엔지니어 취업 면접에서 거의 반드시 나오는 주제다.
2. "에이전트가 검색 도구를 직접 선택한다"는 개념은 최신 LLM 앱 트렌드(Agentic AI)를 이해하는 데 핵심이다.
3. 코드가 공개되어 있고, keyword/semantic/chunk 검색이라는 직관적 구조라 웹 크롤링을 배운 지금 자연스럽게 연결된다.

## 4. Transformer²: Self-adaptive LLMs

**arxiv**: [https://arxiv.org/abs/2501.06252](https://arxiv.org/abs/2501.06252)
**발표일**: 2025년 1월 (2026년 업데이트)
**코드**: ✅ [https://github.com/SakanaAI/self-adaptive-llms](https://github.com/SakanaAI/self-adaptive-llms)

### 초록 요약
LLM이 새로운 태스크에 실시간으로 적응하는 프레임워크 제안. 2-pass 추론 방식으로, 첫 번째 패스에서 태스크 속성을 식별하고 두 번째 패스에서 RL로 학습된 태스크별 전문가 벡터를 동적으로 혼합한다. 가중치 행렬의 특이값(singular value)만 조정하여 LoRA보다 적은 파라미터로 더 높은 성능을 달성했다.

### 추천 이유
1. LoRA를 넘어서는 경량 적응 기법을 제안하며, 파인튜닝의 "다음 단계"를 미리 엿볼 수 있다.
2. Sakana AI(진화 알고리즘으로 유명한 연구소)의 논문으로, 코드와 모델이 모두 공개되어 재현 가능하다.
3. SVD(특이값 분해)라는 선형대수 개념이 LLM에 어떻게 활용되는지 구체적으로 보여주어, 수학 → 실전 연결 감각을 키울 수 있다.
